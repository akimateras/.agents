# 日本語版文書の同期チェックリスト

## 新規翻訳の場合

日本語版がまだ存在しない場合は、変更差分ではなく正本の全体を対象として、このチェックリストを適用する。

- 正本の構成、見出し階層、アンカー、コード例を引き継ぐ。
- 翻訳元としたコミットまたはバージョンを記録する仕組みがあれば、初回から記録する。
- 用語集がない場合は、主要な訳語の対応を最終報告に含め、必要なら用語集の新設を提案する。
- 「同期前」以降の項目のうち、既存の日本語版や変更差分を前提とする項目は適用しない。

## 同期前

- 正本と同期先を特定する。
- 対象となるバージョン、コミット、変更差分を確認する。
- リポジトリの翻訳方針、用語集、同期単位を確認する。
- 日本語版だけに存在する正当な補足や環境差分を特定する。

正本を特定できない場合は、推測で一方を上書きしない。

## 同期時

- 節の追加、変更、削除を差分単位で対応させる。
- 見出し階層、アンカー、リンク先の対応関係を維持する。
- 識別子、コマンド、コード例、UI ラベル、エラーメッセージの原表記を維持する。
- バージョン、既定値、制約、警告を正本と一致させる。
- 正本の記述が実装と矛盾していることに気づいた場合は、同期の範囲では正本に合わせたうえで、矛盾を報告する。正本や日本語版を独断で実装へ合わせない。
- 逐語訳より、技術的な意味と読者が行う操作を一致させる。
- 日本語版固有の補足は、正本との差異であることが分かる形で保持する。
- 正本から削除された内容を残す場合は、残す根拠を確認する。
- 同期した正本のコミットやバージョンを日本語版へ記録する仕組みがあれば、その記録を更新する。

## 同期後の検査

- 正本の各変更に対応する記載がある。
- 見出し、リンク、コード例、表の対応が崩れていない。
- 用語集に従い、同じ概念に同じ訳語を使っている。
- 正式名称以外の未翻訳箇所が意図せず残っていない。
- 日本語版固有の内容を誤って削除していない。
- 同期できなかった箇所と理由を報告している。
